How to get access to /afs/cs.cmu.edu/project/biorobotics

Taken from: http://www.cs.cmu.edu/~help/afs/cross_realm.html

 

1. The first time you try to use Cross Auth:

kinit <username>
aklog cs.cmu.edu

pts createuser <username>@andrew.cmu.edu -cell cs.cmu.edu

 

to verify:

pts examine <username>@andrew.cmu.edu -cell cs.cmu.edu

 

2. Email Matt (mtesch@cmu.edu) with: 1. Andrew ID,  2. Which directory do you need permission.

 

3. In order to edit contents in /afs/cs folder you should install SSH secure shell software.

you can find a copy in myandrew https://www.cmu.edu/myandrew/auth/q?doc=software/alphalist.html

Feb 2011: SSH secure is not avilable any more, you should now probably use SSH Tectia:

https://www.cmu.edu/computing/software/all/tectia/download.html

 

Everytime you run SSH shell and want to reach biorobotics on AFS:

you should open a shell

kinit <username>

aklog cs.cmu.edu

 

then you can go back to the SSH GUI and change directory to your folder /afs/cs.cmu.edu/project/biorobotics/www/...

 

Notes:

kinit is not needed if you are doing this from an andrew unix machine.
Also, you can add the aklog cs.cmu.edu line to your ~/.ssh/rc file and it will happen whenever you log in (this may cause issues with X forwarding.)
